# 需求明确化报告

## 原始需求

帮我搭建一个，干净的接入wdp服务的前端工程，能正常推流，安装好wdp、bim、gis插件，前端ui简单显示下当前的版本信息，以及场景的一些基础空间信息，增加一个按钮回到默认视角的镜头。

## 技术域分类

- **主要域**: `wdp-api-initialization-unified`（工程搭建、SDK 接入、插件安装、推流启动）
- **次要域**:
  - `wdp-api-camera-unified`（回到默认视角按钮）
  - `wdp-api-generic-base-attributes`（版本信息、空间基础信息读取）
  - `wdp-api-function-components`（UI 信息展示面板）

## 需求拆解

### 1. 工程骨架搭建
- 基于 `universal-bootstrap.template.*` 三件套创建项目
- 配置 `package.json` 依赖（CloudApi、WdpApi、BimApi、GisApi）
- 用户配置区集中管理所有核心参数

### 2. 初始化与插件安装
- `new WdpApi()` 实例化
- `App.Plugin.Install()` 安装 WDP、BIM、GIS 三个插件
- `App.Renderer.Start()` 启动渲染器
- 监听 `SceneReady` 事件确认推流成功

### 3. UI 信息展示
- **版本信息**：展示当前 WDP SDK 版本号
- **空间基础信息**：展示场景坐标系、已加载实体数量等基础空间数据
- UI 以简洁卡片形式悬浮在 3D 场景上方

### 4. 默认视角按钮
- 提供一个按钮，点击后调用相机 API 回到场景默认视角
- 按钮位于 UI 面板中或独立悬浮

## 依赖关系

### 技术依赖
- `CloudApi`（SDK 基础）
- `WdpApi`（核心 API）
- `BimApi`（BIM 插件）
- `GisApi`（GIS 插件）

### 时序依赖
```
SDK 加载 → WdpApi 实例化 → Plugin.Install(WDP+BIM+GIS) → Renderer.Start()
→ SceneReady(100%) → 读取版本/空间信息 → 渲染 UI → 绑定按钮事件
```

### 数据依赖
```
SceneReady → GetVersion/GetSceneInfo → UI 渲染
按钮点击 → Camera.Reset/Focus → 视角切换
```
