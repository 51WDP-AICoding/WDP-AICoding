# WDP 场景相机子技能

只负责相机 API 编排，不负责实体查询和页面 UI。

## 前置条件

1. `App` 初始化成功。
2. `Renderer.Start()` 成功。
3. 场景满足 `OnWdpSceneIsReady && progress === 100`。

## 标准流程

1. 读取当前相机状态。
- 记录当前位置/朝向，作为诊断与回退基线。

2. 执行目标相机动作。
- 按需调用 `FlyTo`、`Focus` 或组合调用。

3. 校验动作结果。
- 校验是否到达目标实体或目标视角。

4. 处理失败分支。
- 记录参数摘要和错误对象，必要时回退安全视角。

## 参数规则（2.2.1 基线）

1. 明确角度参数结构。
- 优先使用文档定义的结构体字段（如 `pitch/yaw`）。

2. 控制飞行行为。
- `flyTime`、`distanceFactor` 使用默认值起步，再按体验微调。

3. 串行化相机动作。
- 同一链路避免并发下发多个镜头动作。

## 高频问题

1. 聚焦方向偏移。
- 检查旋转字段表达是否与当前版本一致。

2. 镜头不移动或被打断。
- 检查是否存在并发调用或时序过早。

3. 聚焦失败。
- 检查 eid 有效性与对象可见状态。

## 输出要求

始终输出：
1. 相机调用链路改动点
2. 对用户可见行为的影响
3. 验证步骤与结果
4. 回退策略（如有）
