# WDP 通用事件注册子技能

只负责事件层，避免在本 skill 中混入大段业务逻辑。

## 前置条件

1. `App` 已实例化。
2. `Renderer.Start()` 已成功。
3. 场景相关事件在合适时机注册。

## 标准流程

1. 定义事件清单。
- 明确事件名、用途、成功条件、失败分支。

2. 统一注册入口。
- 只在一个入口注册，避免多处重复绑定。

3. 在回调中先做守卫。
- 先校验 `res` 结构，再执行业务分发。

4. 输出结构化日志。
- 至少记录 `eventName`、关键参数摘要、结果、错误对象。

5. 维护解绑机制。
- 路由切换或重建场景时回收旧监听。

## 高频问题

1. 事件不触发。
- 检查注册时机、事件名和渲染启动状态。

2. 事件重复触发。
- 检查是否重复注册且未解绑。

3. 回调空对象报错。
- 检查回调守卫和字段容错。

## 输出要求

始终输出：
1. 事件注册或回收改动点
2. 稳定性影响
3. 修复补丁或实现建议
4. 验证步骤与预期
